도구는 에이전트가 실행할 수 있는 단일 작업이며, 서비스 호출이나 등록된 operation 실행처럼 구체적인 액션을 담당합니다. Moldy의 도구 화면은 등록된 도구와 실행 가능 상태를 보여주고, 에이전트 설정 화면은 이 도구 중 필요한 항목을 에이전트에 연결합니다.

이 문서는 catalog 준비 상태와 런타임 사용을 구분하는 데 사용합니다. 도구가 catalog에 있어도 에이전트에 연결되고 필요한 자격증명이 준비되기 전까지는 해당 에이전트가 사용할 수 없습니다.

## 도구와 MCP 도구의 차이

| 종류 | 어디서 관리하나 | 에이전트에 연결되는 방식 |
| --- | --- | --- |
| 내장/등록 도구 | **도구** 화면 | 에이전트 설정의 도구 선택 목록 |
| MCP 도구 | **MCP 서버** 화면에서 발견 | 에이전트 설정의 MCP 도구 선택 목록 |
| 스킬 | **스킬** 화면 | 에이전트 설정의 스킬 선택 목록 |

도구 화면에 도구가 보이는 것과 에이전트가 실제로 사용할 수 있는 것은 다릅니다. 에이전트 설정에서 해당 도구를 연결해야 런타임에 전달됩니다.

## 도구 사용 전 점검

1. 도구가 요구하는 자격증명 유형을 확인합니다.
2. 사용자 자격증명 또는 시스템 자격증명이 준비되어 있는지 확인합니다.
3. 에이전트 설정에서 도구를 연결합니다.
4. 테스트 채팅으로 도구 호출이 발생하는지 확인합니다.
5. 실패하면 채팅 결과의 도구 오류와 자격증명 테스트 결과를 함께 확인합니다.

## 도구 실행 결과 읽기

채팅 화면은 도구 호출 결과를 대화 흐름 안에서 보여줍니다. 실패한 경우에는 다음 순서로 확인하세요.

- 도구가 에이전트에 연결되어 있는지
- 필요한 자격증명이 현재 사용자에게 있는지
- MCP 도구라면 원본 MCP 서버 상태가 정상인지
- 모델이 도구 호출을 지원하는지

## 운영자와 일반 사용자 범위

일반 사용자는 자신의 에이전트에 연결할 도구와 사용자 자격증명을 관리합니다. 운영자는 시스템 자격증명과 System LLM을 관리하지만, 사용자 소유 자격증명 값은 공개 문서나 캡처에 노출하지 않아야 합니다.

도구를 문서화할 때는 필요한 자격증명 유형과 실패 가능성을 설명하되 credential 값은 포함하지 마세요. workflow에서 사용할 수 있다고 보기 전에 테스트 채팅에서 도구를 검증하세요.
