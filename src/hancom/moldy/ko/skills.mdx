스킬은 에이전트가 특정 작업을 더 잘 수행하도록 돕는 재사용 가능한 지식과 파일 묶음입니다. 스킬은 지침, 텍스트 콘텐츠, 파일, 패키지 asset, 자격증명 요구사항을 포함할 수 있고 에이전트에 연결됩니다.

Moldy는 스킬 생성, 업로드, 파일 관리, 콘텐츠 편집, 자격증명 요구사항과 바인딩 API를 제공합니다. 에이전트 지침을 더 늘릴지, 새 도구를 등록할지, 스킬을 만들지 결정할 때 이 문서를 기준으로 사용하세요.

## 스킬이 하는 일

스킬은 도구와 다릅니다. 도구는 실행 가능한 액션이고, 스킬은 에이전트가 참고하거나 실행에 사용할 지식/파일/패키지입니다.

| 항목 | 스킬 | 도구 |
| --- | --- | --- |
| 목적 | 지식, 지침, 파일 패키지 제공 | 외부 작업 실행 |
| 에이전트 연결 | 에이전트 설정의 스킬 목록 | 에이전트 설정의 도구 또는 MCP 도구 목록 |
| 자격증명 | 스킬 요구사항에 맞춰 바인딩 가능 | 도구 정의 또는 MCP 서버 설정에서 요구 |
| 마켓플레이스 | skill package로 게시 가능 | 직접 게시보다는 스킬/MCP 리소스에 포함 |

## 스킬 생성과 업로드

Moldy API에는 스킬을 직접 생성하는 경로와 패키지를 업로드하는 경로가 있습니다. 업로드 또는 게시 전에는 secret scan이 실행되어 패키지 안의 잠재적인 비밀값을 막습니다.

1. **스킬** 화면에서 새 스킬을 만듭니다.
2. 이름과 설명을 입력합니다.
3. 텍스트 콘텐츠 또는 파일을 추가합니다.
4. 필요한 경우 자격증명 요구사항을 정의합니다.
5. 에이전트 설정에서 스킬을 연결합니다.

## 파일 관리

스킬은 파일 목록 조회, 파일 내용 조회, 파일 추가/수정/삭제 API를 갖습니다. 이 구조 덕분에 단순 텍스트뿐 아니라 여러 파일로 구성된 지식 패키지도 관리할 수 있습니다.

파일 기반 스킬은 에이전트가 안정적인 참조 자료를 필요로 할 때 유용합니다. 파일은 좁은 범위로 유지하고, 오래된 내용을 제거하고, 새 마켓플레이스 버전을 게시하기 전에 스킬을 갱신하세요.

<Tip>
스킬 파일에는 실제 API 키, 개인 계정 정보, 내부 URL을 넣지 마세요. 마켓플레이스 게시 흐름에서도 secret scan이 비밀값을 차단합니다.
</Tip>

## 자격증명 요구사항과 바인딩

스킬은 실행에 필요한 자격증명 요구사항을 선언할 수 있습니다. 사용자는 자신의 자격증명 중 요구사항과 맞는 항목을 바인딩합니다.

| 흐름 | 설명 |
| --- | --- |
| 요구사항 조회 | 스킬이 어떤 credential definition을 요구하는지 확인 |
| 바인딩 조회 | 현재 사용자에게 연결된 자격증명 확인 |
| 바인딩 저장 | 요구사항 key와 사용자 자격증명을 연결 |
| 바인딩 삭제 | 더 이상 쓰지 않는 연결 제거 |

서버는 다른 사용자의 자격증명이나 시스템 자격증명을 스킬 바인딩에 잘못 연결하지 못하도록 검증합니다.

## 에이전트에서 사용하기

에이전트 설정에서 스킬을 선택하면 채팅 런타임이 해당 스킬 정보를 에이전트 구성에 포함합니다. 채팅 화면은 연결된 스킬을 표시하고, 에이전트는 필요한 경우 스킬 내용을 활용합니다.

연결된 스킬은 모델 선택, 자격증명 설정, 도구 연결을 대체하지 않습니다. 테스트 채팅으로 에이전트가 의도한 지식을 읽고 필요한 자격증명을 예상대로 요구하는지 확인하세요.
