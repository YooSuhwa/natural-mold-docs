운영자 설정은 일반 사용자가 에이전트를 만들기 전에 Moldy의 공용 실행 환경을 준비하는 절차입니다. 시스템 자격증명, System LLM 설정, 모델 상태가 먼저 준비되어야 에이전트 생성, 도구 실행, 스케줄, 사용량 집계가 안정적으로 이어집니다.

이 문서는 새 환경의 준비 상태를 확인하는 checklist입니다. 일반 사용자가 에이전트를 안정적으로 만들려면 운영자 전용 플랫폼 의존성이 먼저 설정되고 테스트되어야 합니다.

```mermaid
flowchart LR
  A["운영자 계정 확인"] --> B["시스템 자격증명 등록"]
  B --> C["System LLM 설정"]
  C --> D["모델 상태 확인"]
  D --> E["첫 에이전트 생성"]
  E --> F["사용량과 스케줄 점검"]
```

## 운영자 권한 확인

1. 관리자 계정으로 로그인합니다.
2. 사이드바에서 시스템 설정, 시스템 자격증명, System LLM 메뉴가 보이는지 확인합니다.
3. 메뉴가 보이지 않으면 계정의 운영자 권한 또는 배포 환경의 초기 관리자 설정을 먼저 확인합니다.

## 시스템 자격증명 등록

<img className="moldy-screenshot" src="/images/hancom/moldy/ko/system-credentials.png" alt="Moldy 시스템 자격증명 화면" />

시스템 자격증명은 공용 모델, MCP 서버, 내장 도구가 사용할 비밀값을 운영자 범위에서 관리합니다.

1. **시스템 자격증명** 메뉴를 엽니다.
2. 공급자 또는 서비스에 맞는 자격증명을 추가합니다.
3. 이름, 설명, 키 값을 입력하고 저장합니다.
4. 사용자에게 노출되어도 되는 설명과 실제 비밀값을 분리해서 관리합니다.

시스템 자격증명 이름은 운영자가 선택할 수 있을 만큼 명확해야 하지만 secret 값을 노출하면 안 됩니다. 화면 캡처와 공개 문서에는 상태와 설정 구조만 보여주세요.

## System LLM 설정

<img className="moldy-screenshot" src="/images/hancom/moldy/ko/system-llm.png" alt="Moldy System LLM 설정 화면" />

System LLM은 추천, 템플릿 보조, 일부 운영 기능처럼 사용자 에이전트와 분리된 시스템 작업에 쓰입니다.

1. **System LLM** 메뉴를 엽니다.
2. 기본 제공자와 모델을 선택합니다.
3. 필요한 경우 시스템 자격증명을 연결합니다.
4. 저장 후 모델 호출이 가능한지 상태를 확인합니다.

## 첫 에이전트로 검증

설정이 끝나면 간단한 수동 에이전트를 만들어 채팅을 실행합니다. 응답이 정상적으로 생성되고, 사용량 페이지에 호출 기록이 남고, 운영자 전용 설정 누락이 없다면 운영자 초기 설정은 완료된 상태로 볼 수 있습니다.

검증에 실패하면 Builder와 Assistant 문제는 System LLM 설정을 먼저 확인하고, 일반 채팅 런타임 문제는 사용자 자격증명을 먼저 확인하세요.
