---

System LLM 설정은 Moldy 플랫폼 workflow에 사용할 운영자 전용 모델 슬롯입니다. Builder, Assistant, 이미지 생성이 사용하는 시스템 모델을 역할별로 선택하며, 일반 에이전트 채팅에서 사용하는 사용자 소유 모델 자격증명과 분리됩니다.

대화형 에이전트 생성, Fix 지원, 에이전트 이미지 생성을 안정적으로 사용하기 전에 이 설정을 완료하세요. 일반 채팅은 동작하지만 플랫폼 workflow가 실패한다면 System LLM 설정을 먼저 확인합니다.

<img className="moldy-screenshot" src="/images/hancom/moldy/ko/system-llm.png" alt="Moldy 시스템 LLM 설정 화면" />

## 역할 슬롯

| 슬롯 | 용도 |
| --- | --- |
| 텍스트 기본 모델 | Builder와 Assistant가 기본적으로 사용하는 텍스트 LLM |
| 텍스트 폴백 모델 | 기본 모델 호출이 실패할 때 사용하는 텍스트 LLM |
| 이미지 모델 | 이미지 생성에 사용하는 모델 |

각 슬롯은 운영자가 테스트한 모델과 자격증명을 가리켜야 합니다. 슬롯이 비어 있거나 맞지 않으면 사용자 에이전트에 동작하는 자격증명이 있어도 Builder, Assistant, 이미지 생성이 실패할 수 있습니다.

## 설정 절차

1. **시스템 자격증명**에서 LLM 자격증명을 등록합니다.
2. **시스템 LLM 설정**으로 이동합니다.
3. 각 슬롯에서 시스템 자격증명과 모델 이름을 선택합니다.
4. 저장 후 Builder 또는 Assistant 흐름에서 동작을 확인합니다.

System LLM 설정을 사용자 채팅 자격증명의 대체 수단으로 사용하지 마세요. 일반 에이전트 대화는 선택한 모델을 호출할 수 있는 사용자 소유 자격증명 또는 런타임에서 허용된 별도 자격증명 경로가 필요합니다.

<Warning>
세 슬롯이 모두 설정되지 않으면 Builder, Assistant, 이미지 생성 기능이 기대대로 동작하지 않을 수 있습니다.
</Warning>
