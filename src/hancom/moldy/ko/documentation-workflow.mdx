Moldy 문서는 현재 코드, 실행 중인 OpenAPI 스키마, 실제 화면 캡처를 기준으로 유지합니다. 프론트엔드 라우트, API, 권한, UI 문구가 바뀌면 문서 산출물도 같은 변경 흐름에서 갱신하세요.

이 워크플로는 한국어와 영어 문서를 제품과 맞게 유지하면서 사용자와 AI 시스템이 각 페이지를 더 쉽게 인용할 수 있게 합니다. 모든 제품 관련 사실은 route, API schema, source file, screenshot, 브라우저로 확인한 화면 중 하나로 추적 가능해야 합니다.

## 갱신 순서

1. Moldy 앱과 API를 실행합니다.
2. OpenAPI 스냅샷을 갱신합니다.
3. 기능 인벤토리를 다시 생성합니다.
4. Playwright 캡처를 다시 실행합니다.
5. 한국어와 영어 문서를 함께 수정합니다.
6. 민감정보, 이미지 참조, 링크를 검증합니다.
7. 문서 서버에서 핵심 페이지를 확인합니다.

```bash
pnpm run openapi
pnpm run inventory
pnpm run capture
pnpm run check:sensitive
pnpm run docs:check
```

## 변경 시 확인할 것

| 변경 유형 | 확인 항목 |
| --- | --- |
| 새 화면 또는 라우트 | `planning/feature-inventory.md`의 문서화 여부와 캡처 필요 여부 |
| 새 API | API Reference 탭과 `src/openapi/moldy-openapi.json` |
| 권한 변경 | 사용자/운영자 접근 조건과 문제 해결 문서 |
| UI 문구 변경 | 한국어/영어 캡처와 본문 표현 |
| 민감정보가 보이는 화면 | 캡처 마스킹 규칙과 `check:sensitive` 결과 |

## 소스 근거 확인

외부 제품 문서는 정보 구조 참고용입니다. Moldy 문서에 기능을 추가하기 전에는 다음 위치에서 실제 구현 여부를 확인합니다.

| 근거 | 예시 |
| --- | --- |
| 프론트 라우트 | `natural-mold/frontend/src/app/**/page.tsx` |
| 백엔드 API | `natural-mold/backend/app/routers/*.py` |
| API 스키마 | `src/openapi/moldy-openapi.json` |
| 테스트 | `natural-mold/frontend/tests/pages/*.test.tsx` |
| 실제 화면 | `src/images/hancom/moldy/{ko,en}` |

현재 문서의 마켓플레이스 설치/게시, MCP probe/import/export, 공유 링크, 스케줄, 채팅 스트리밍 내용은 Moldy 소스에서 확인된 라우트와 API만 기준으로 작성합니다.

## GEO와 AI 가시성 점검

큰 기능 추가, 메뉴 변경, API 변경, 문서 구조 변경 후에는 GEO/SEO 관점도 함께 확인합니다. 목표는 사용자가 검색 엔진이나 AI 답변에서 Moldy 기능을 찾았을 때, 각 페이지가 정확한 답변 단위로 인용될 수 있게 만드는 것입니다.

| 점검 항목 | 기준 |
| --- | --- |
| Answer-first 구조 | 첫 문단이 페이지의 목적, 대상 기능, 사용자가 얻는 결과를 바로 설명해야 합니다. |
| 인용 가능한 단위 | 기능 설명, 절차, 상태, 제한, 문제 해결을 짧은 heading, 표, 단계로 나눕니다. |
| `llms.txt` | 주요 메뉴, 채팅 인라인 컴포넌트, 파일/아티팩트 뷰어, API, 운영 문서가 추가되면 `src/llms.txt` 링크와 설명을 갱신합니다. |
| `robots.txt` | 배포 정책상 제한이 없다면 문서 크롤링을 허용하고 `/llms.txt` 위치를 안내합니다. |
| `sitemap.xml` | production 문서 도메인이 확정된 뒤 absolute canonical URL로 생성합니다. 임시 URL이나 placeholder는 게시하지 않습니다. |
| JSON-LD | production URL, frontmatter, navigation path, 언어, 수정일을 알 수 있을 때 build-time metadata로 주입합니다. |
| 변경 기록 | GEO 점검 결과와 남은 production 작업은 `planning/` 리포트와 변경 이력에 남깁니다. |

## 구조화 데이터 권장안

문서 사이트의 구조화 데이터는 MDX 본문에 raw `<script>` 예제를 직접 넣는 방식이 아니라, 구현 수준의 메타데이터로 게시하는 것을 권장합니다. 문서 프레임워크가 build-time JSON-LD 주입을 지원한다면 최종 page frontmatter, production URL, navigation path, screenshot asset을 기준으로 생성하세요.

| 페이지 유형 | 권장 Schema.org type | 값의 출처 |
| --- | --- | --- |
| 문서 홈 | `WebSite`와 `SoftwareApplication` | 공개 문서 URL, Moldy 제품명, 최종 설명, 검색 URL이 있으면 검색 URL |
| 기술 가이드 | `TechArticle` | MDX `title`, MDX `description`, canonical page URL, 언어, 수정일 |
| 문제 해결 또는 FAQ형 페이지 | 명시적인 질문/답변이 있을 때 `FAQPage` | 최종 질문 heading과 답변 본문 |
| 탐색 경로 | `BreadcrumbList` | `src/docs.json`의 navigation path와 canonical page URL |

다음 JSON-LD는 참고 템플릿입니다. 사이트 생성 시 placeholder 문자열을 최종 production 값으로 바꿔 사용하세요.

```json
{
  "@context": "https://schema.org",
  "@graph": [
    {
      "@type": "SoftwareApplication",
      "@id": "PUBLIC_DOCS_BASE_URL/#software",
      "name": "Moldy",
      "applicationCategory": "BusinessApplication",
      "description": "Moldy is a web application for building and operating AI agents through chat, templates, manual configuration, tools, MCP servers, skills, schedules, and marketplace resources.",
      "url": "PUBLIC_DOCS_BASE_URL/hancom/moldy/ko"
    },
    {
      "@type": "WebSite",
      "@id": "PUBLIC_DOCS_BASE_URL/#website",
      "name": "Moldy documentation",
      "url": "PUBLIC_DOCS_BASE_URL",
      "about": {
        "@id": "PUBLIC_DOCS_BASE_URL/#software"
      },
      "inLanguage": ["ko", "en"]
    },
    {
      "@type": "TechArticle",
      "@id": "PAGE_URL#article",
      "headline": "PAGE_TITLE",
      "description": "PAGE_DESCRIPTION",
      "url": "PAGE_URL",
      "inLanguage": "ko",
      "about": {
        "@id": "PUBLIC_DOCS_BASE_URL/#software"
      },
      "isPartOf": {
        "@id": "PUBLIC_DOCS_BASE_URL/#website"
      },
      "dateModified": "YYYY-MM-DD"
    }
  ]
}
```

placeholder 값은 게시하지 마세요. 배포 후 생성된 JSON-LD를 검증하고, navigation model, canonical URL 전략, page metadata가 바뀌면 템플릿도 함께 갱신하세요.

## 로컬 문서 서버

문서 사이트는 Moldy 앱과 분리해서 실행합니다.

```bash
pnpm run docs:dev
```

기본 확인 주소는 `http://localhost:3001`입니다.
