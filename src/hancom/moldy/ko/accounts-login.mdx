Moldy 계정은 브라우저 애플리케이션에서 이메일과 비밀번호 기반 인증을 사용합니다. 로그인 후 인증된 사용자는 자신의 에이전트, 대화, 자격증명, 도구 구성을 관리할 수 있고, `super_user` 권한이 있는 사용자는 운영자 설정에도 접근할 수 있습니다.

로그인 상태는 백엔드 세션으로 검증되고 권한별 메뉴에 반영됩니다. 사용자가 로그인은 되지만 시스템 설정이 보이지 않는다면 일반 로그인 문제가 아니라 운영자 권한 문제로 먼저 구분하세요.

<img className="moldy-screenshot" src="/images/hancom/moldy/ko/login.png" alt="이메일과 비밀번호 필드가 있는 Moldy 로그인 화면" />

## 로그인

1. `/login`으로 이동합니다.
2. 이메일과 비밀번호를 입력합니다.
3. **로그인**을 선택합니다.
4. 성공하면 대시보드로 이동합니다.

로그인이 성공하면 브라우저 세션이 만들어지고, `/api/auth/me`가 현재 사용자를 반환하며, 대시보드가 표시되어야 합니다. 이 중 하나라도 실패하면 [문제 해결](/hancom/moldy/ko/troubleshooting)에서 쿠키, CSRF, API base URL 설정을 확인하세요.

## 계정 만들기

`/register`에서 새 계정을 만들 수 있습니다. 개발 환경에서는 설정에 따라 첫 번째 가입자가 운영자 권한을 받을 수 있습니다. 운영 환경에서는 첫 운영자 정책을 배포 전에 확정하고, 일반 사용자가 의도치 않게 운영자가 되지 않도록 설정을 점검하세요.

<img className="moldy-screenshot" src="/images/hancom/moldy/ko/register.png" alt="이름, 이메일, 비밀번호와 약관 동의가 있는 Moldy 회원가입 화면" />

## 권한별 메뉴

| 권한 | 보이는 주요 메뉴 |
| --- | --- |
| 일반 사용자 | 홈, 새 에이전트, 도구, MCP 서버, 스킬, 마켓플레이스, 자격증명, 모델, 스케줄, 사용량 |
| 운영자 | 일반 사용자 메뉴 + 시스템 자격증명, 시스템 LLM 설정, 마켓플레이스 운영자 관리 |

메뉴 노출은 사용성을 위한 신호일 뿐 유일한 보호 경계가 아닙니다. 운영자 API는 백엔드에서도 `super_user`를 검증하고, 상태를 변경하는 API 요청은 인증 세션과 CSRF 검증을 통과해야 합니다.

<Warning>
문서나 화면 캡처에는 실제 이메일, 비밀번호, API 키, 내부 URL을 포함하지 마세요. 이 문서의 이미지는 캡처 전에 예시 데이터로 마스킹합니다.
</Warning>
