Moldy artifact viewer는 파일의 `kind`, extension, MIME type, status를 조합해 preview provider를 선택합니다. 이 페이지는 **어떤 파일이 어떤 뷰어로 열리는지**를 확인하는 레퍼런스입니다.

파일 라이브러리 사용법, 즐겨찾기, 다운로드, 공유 흐름은 [파일과 아티팩트](/hancom/moldy/ko/files-artifacts)를 참고하세요.

## 전체 매트릭스

| 파일/콘텐츠 | 판정 신호 | 선택되는 뷰어 | 주요 확인 | fallback/주의 |
| --- | --- | --- | --- | --- |
| 이미지 | `image/*`, `png`, `jpg`, `jpeg`, `webp` | 이미지 preview | 확대, 시각 정보, 다운로드 | 공유 전 이미지 안의 개인정보 확인 |
| 오디오 | `audio/*`, `mp3`, `wav` | 브라우저 audio player | 재생 가능 여부, 길이 | 브라우저 codec 지원 필요 |
| 비디오 | `video/*`, `mp4`, `webm` | 브라우저 video player | 재생, poster, 크기 | 큰 파일은 download가 더 안정적 |
| PDF | `application/pdf`, `pdf` | PDF embed | page, 문서 내용 | 브라우저 PDF 정책에 영향 |
| HTML | `text/html`, `html`, `htm` | sandboxed iframe | 렌더링 결과, 링크 | 외부 링크와 embedded media 확인 |
| Markdown | `text/markdown`, `md`, `markdown` | Markdown renderer | heading, table, link | raw text fallback |
| Mermaid | `mmd`, `mermaid` | Mermaid renderer | flowchart, sequence | 문법 오류 시 코드로 확인 |
| CSV/TSV | `csv`, `tsv`, text table | table preview | delimiter, header, row count | 너무 큰 파일은 일부만 확인 |
| JSON | `application/json`, `json` | structured JSON preview | key, array/object 깊이 | 민감 field 확인 |
| YAML/TOML | `yaml`, `yml`, `toml` | key/value preview | 설정값, placeholder | secret-like 값 확인 |
| Code | artifact kind `code` | code preview | language, path, overwrite 위험 | 실행 전 별도 검증 |
| TXT/LOG | `text/*`, `txt`, `log` | text preview | error line, stack trace | 내부 경로/토큰 확인 |
| DOCX | `docx` | document preview | 본문, 표, 제목 | download fallback |
| XLS/XLSX | `xls`, `xlsx` | sheet/table preview | sheet, row, column | 수식/숨김 sheet 확인 |
| PPTX | `pptx` | slide preview | slide 순서, 본문 | download fallback |
| HWP/HWPX | `hwp`, `hwpx` | 한글 문서 preview | 문단, 표, 본문 구조 | download fallback |
| 기타/CAD | `cad`, `dwg`, 알 수 없는 MIME | fallback card | 파일명, 크기, download | preview provider 추가 여부 검토 |

## 선택 기준

| 기준 | 사용 방식 |
| --- | --- |
| `kind` | `image`, `video`, `audio`, `pdf`, `markdown`, `html`, `code`, `document`, `data`, `cad`, `other` 같은 상위 분류 |
| Extension | 실제 파일명 기반 판정. MIME type이 불완전할 때 provider 선택에 중요 |
| MIME type | `image/*`, `text/html`, `application/pdf`, `application/json` 같은 content type |
| Status | `ready`일 때 preview/download 가능. `writing`, `failed`, `deleted`는 별도 상태로 처리 |
| Size limit | text preview byte limit, maximum artifact bytes, maximum files per run의 영향을 받음 |

## 위치별 차이

| 위치 | 가장 잘 보는 것 |
| --- | --- |
| 채팅 artifact card | 방금 생성된 파일과 실행 맥락 |
| 오른쪽 artifact panel | 같은 대화의 파일 목록과 preview |
| **Settings > Files** | 여러 에이전트/대화의 파일 검색, 필터, favorite, stats |
| 공개 공유 화면 | share token으로 노출되는 read-only snapshot과 공유 파일 |

## 실패 판정

| 증상 | 우선 확인 |
| --- | --- |
| preview가 비어 있음 | artifact status, MIME type, extension, size limit |
| 다운로드가 안 됨 | storage object 존재, current version, owner permission |
| 공유 화면에서 안 열림 | active share token, artifact가 공유 대화에 속하는지 |
| 문서가 fallback만 보임 | 해당 extension/MIME provider 지원 여부 |
| 오래된 파일이 보임 | version number와 update time |

<Warning>
HTML preview는 sandboxed iframe으로 열립니다. 그래도 외부 링크, embedded media, 민감한 본문이 포함될 수 있으므로 공유 전 내용을 확인하세요.
</Warning>

## 운영 체크리스트

파일 viewer 기능이 추가되거나 변경되면 다음을 같이 갱신하세요.

1. 이 매트릭스의 extension/MIME type 행.
2. [파일과 아티팩트](/hancom/moldy/ko/files-artifacts)의 요약 표.
3. [채팅 컴포넌트 갤러리](/hancom/moldy/ko/chat-components)의 artifact 카드 설명.
4. 공개 공유와 보안 주의.
5. `src/llms.txt`의 Files/artifacts 설명.
