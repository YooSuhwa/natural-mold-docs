**Settings > Usage** 화면은 대화 실행에서 수집한 토큰, 요청 수, 추정 비용을 기간별로 보여줍니다. 어떤 에이전트와 모델이 런타임 자원을 사용하고 있는지 이해하는 데 사용합니다.

백엔드에는 summary, daily spend, agent usage API가 있으며, 사용자 소유 범위를 기준으로 집계합니다. 비용은 모델 catalog의 가격 메타데이터에 의존하므로, 별도 외부 billing system과 대조하지 않는 한 사용량 보고서는 모델 실행 기준 추정치로 읽어야 합니다.

## 주요 지표

| 지표 | 의미 |
| --- | --- |
| 비용 | 모델 가격 메타데이터와 사용량을 기반으로 계산한 추정 비용 |
| 입력 토큰 | prompt tokens |
| 출력 토큰 | completion tokens |
| 요청 수 | 모델 호출 또는 대화 실행 횟수 |
| 평균 비용/요청 | 비용을 요청 수로 나눈 값 |

채팅 메시지 안에서도 token usage popover를 통해 prompt, completion, cache token, estimated cost를 확인할 수 있습니다. 한 번의 실행은 chat-level popover에서 보고, 집계 조사는 Usage 화면에서 확인하세요.

## 집계 기준

`/api/usage/daily`는 `target_kind`를 기준으로 집계합니다.

| target_kind | 설명 |
| --- | --- |
| `user` | 현재 사용자 기준 |
| `agent` | 사용자의 에이전트 기준 |
| `model` | 사용자의 에이전트가 호출한 모델 기준 |

`group_by=date`는 날짜별 합계를, `group_by=target`은 날짜와 target을 함께 나누어 다중 시리즈 차트에 사용할 수 있는 데이터를 반환합니다.

## 사용량을 볼 때 주의할 점

- 비용은 모델 catalog의 가격 메타데이터에 의존합니다.
- 모델 가격이 비어 있거나 잘못되어 있으면 비용 계산도 부정확할 수 있습니다.
- trace나 usage 원천 데이터가 없는 과거 메시지는 집계가 제한될 수 있습니다.
- 스케줄 실행도 에이전트 실행이므로 사용량에 반영될 수 있습니다.

사용량 데이터는 실행을 만든 에이전트, 모델, 스케줄과 함께 볼 때 가장 유용합니다. 갑작스러운 비용 변화는 요약 차트만으로 판단하지 말고 대화 기록과 스케줄 기록을 함께 확인하세요.

## 데이터 내려받기

화면의 CSV 다운로드를 사용하면 표시된 기간과 그룹 기준의 원본 데이터를 받을 수 있습니다. 정산이나 이상 사용량 분석에는 CSV와 모델별 상세 내역을 함께 확인하세요.

## 이상 사용량 점검

1. 기간을 최근 7일 또는 문제 발생 기간으로 좁힙니다.
2. 에이전트별 비용을 확인합니다.
3. 비용이 높은 에이전트의 대화와 스케줄을 확인합니다.
4. 모델 fallback이 반복 실패를 만들지 않는지 확인합니다.
5. 필요하면 max_tokens, 스케줄 주기, 도구 사용 조건을 조정합니다.
