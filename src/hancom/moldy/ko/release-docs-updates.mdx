릴리즈와 문서 업데이트 문서는 제품 변경을 문서에 반영하는 운영 흐름입니다. 최근 커밋, 메뉴 변경, OpenAPI 변경, 스크린샷, GEO/SEO 자산, 변경 이력을 함께 보면서 문서가 제품과 어긋나지 않게 유지합니다.

실제 작업자용 상세 규칙은 `AGENTS.md`에 있고, 이 페이지는 문서 사이트 안에서 볼 수 있는 사용자/운영자용 요약입니다.

## 언제 업데이트할까

| 변경 | 문서 액션 |
| --- | --- |
| 새 메뉴 또는 라우트 | navigation, quickstart, settings, screenshot 업데이트 |
| 채팅 컴포넌트 변경 | 채팅 가이드, 컴포넌트 갤러리, 오류 사전 업데이트 |
| artifact viewer 변경 | 파일 가이드, 뷰어 매트릭스, 보안 문서 업데이트 |
| API router/schema 변경 | OpenAPI, API endpoint index, API cookbook 업데이트 |
| 권한/운영자 기능 변경 | 접근 권한, 보안, FAQ, 문제 해결 업데이트 |
| 스케줄/Agent API 변경 | 레시피, API cookbook, 오류 사전 업데이트 |
| 문서 구조 변경 | `llms.txt`, changelog, GEO 리포트 업데이트 |

## 최근 커밋에서 문서 업데이트하기

제품 소스는 `/Users/chester/dev/ref/natural-mold`에 있습니다. 최근 변경을 볼 때는 commit subject만 믿지 말고 변경 파일과 실제 source를 확인합니다.

```bash
git -C /Users/chester/dev/ref/natural-mold log --oneline --decorate -20
git -C /Users/chester/dev/ref/natural-mold show --stat --name-only <commit>
git -C /Users/chester/dev/ref/natural-mold show --name-status --format=fuller <commit>
```

변경이 broad하면 route, router, schema, screenshot 범위를 함께 봅니다.

```bash
rg -n "새 기능 이름|메뉴 라벨|endpoint" /Users/chester/dev/ref/natural-mold/frontend/src /Users/chester/dev/ref/natural-mold/backend/app
```

## 갱신 산출물

| 산출물 | 언제 갱신 |
| --- | --- |
| `src/docs.json` | 페이지, 메뉴, 탭, 언어 구조 변경 |
| KO/EN MDX | 사용자 가이드와 reference 변경 |
| `src/openapi/moldy-openapi.json` | backend endpoint 또는 schema 변경 |
| `api-endpoints.mdx` | OpenAPI snapshot 변경 |
| `planning/feature-inventory.md` | route나 기능 범위 변경 |
| screenshots | UI label, layout, visible state 변경 |
| 실제 UI reference capture | 채팅 컴포넌트나 artifact viewer 화면 변경 |
| `src/llms.txt` | 주요 문서, API, 레시피, viewer, FAQ 변경 |
| changelog | 사용자 안내에 영향을 주는 문서 변경 |

## 같이 바뀌어야 하는 문서

| 변경 | 함께 볼 문서 |
| --- | --- |
| 채팅 UI 또는 인라인 카드 | 채팅 가이드, 채팅 컴포넌트 갤러리, 오류 메시지 사전 |
| 파일/아티팩트 viewer | 파일 가이드, 아티팩트 뷰어 매트릭스, 보안과 프라이버시 |
| 채팅/파일 실제 UI 캡처 | `scripts/capture-ui-reference-images.mjs`, 이미지 alt text, 변경 이력 |
| Agent API 또는 `/v1` runtime | Agent API, API Cookbook, API reference, 오류 메시지 사전 |
| 권한, 공유, 자격증명 | 접근 권한과 감사, 보안과 프라이버시, FAQ |
| 반복 질문이나 지원 이슈 | 문제 해결, 오류 메시지 사전, FAQ |
| 문서 구조나 발견성 | `src/docs.json`, `src/llms.txt`, 변경 이력 |

## 검증 순서

```bash
pnpm run docs:check
pnpm run check:sensitive
```

API가 바뀌었다면 다음을 먼저 실행합니다.

```bash
pnpm run openapi
pnpm run api-docs
pnpm run docs:check
```

화면이 바뀌었다면 캡처를 다시 만들고 민감정보를 확인합니다.

```bash
pnpm run capture
pnpm run check:sensitive
```

채팅 컴포넌트나 artifact viewer의 실제 UI 캡처가 바뀌었다면 다음을 실행합니다.

```bash
pnpm run capture:ui
pnpm run check:sensitive
```

## 문서 품질 체크

| 질문 | 기준 |
| --- | --- |
| 메뉴 이름이 실제 UI와 같은가 | screenshot 또는 route 기준으로 확인 |
| 기능이 source에 있는가 | frontend route, backend router, OpenAPI, test 중 하나로 확인 |
| KO/EN이 같은 사실을 말하는가 | 구조와 핵심 표를 맞춤 |
| 사용자가 다음 행동을 알 수 있는가 | 단계, 실패 확인, 관련 링크 포함 |
| AI가 인용하기 쉬운가 | answer-first 문단, 짧은 heading, 표, `llms.txt` |

## 변경 이력 작성 기준

변경 이력에는 제품 릴리즈 노트가 아니라 문서 변경을 기록합니다. 다음 내용을 남기면 좋습니다.

- 새로 추가된 페이지.
- 기능 설명이 확장된 페이지.
- OpenAPI endpoint 수나 API 그룹 변경.
- 스크린샷 범위 변경.
- GEO/SEO, `llms.txt`, GitHub 링크 같은 discovery 자산 변경.
