Moldy 채팅 화면은 텍스트 답변뿐 아니라 Markdown, 코드, 다이어그램, 수식, 이미지, 검색 결과, 도구 실행, 승인 요청, 사용자 입력, 메모리 제안, 서브에이전트, Builder 단계, 생성 파일을 전용 컴포넌트로 표시합니다.

실제 대화 흐름은 [에이전트와 채팅하기](/hancom/moldy/ko/chat-with-agent)를, 스트리밍 이벤트 구조는 [채팅 스트리밍 API](/hancom/moldy/ko/api-chat-streaming)를 참고하세요. 이 페이지는 **채팅에 나타난 UI가 무엇인지 빠르게 식별하는 레퍼런스**입니다.

## 컴포넌트 매트릭스

| 컴포넌트 | 나타나는 상황 | 사용자가 할 일 | 주의 |
| --- | --- | --- | --- |
| Markdown 본문 | 일반 답변, 설명, 요약 | heading, 표, 링크를 읽고 다음 요청 작성 | 스트리밍 중 표가 임시로 깨질 수 있음 |
| 코드 블록 | 코드, 로그, 설정 예시 | 언어 라벨과 내용을 확인 | 실행 가능한 코드인지 별도 검증 필요 |
| Mermaid | `mermaid` 코드 블록 | 다이어그램 구조 확인 | 문법 오류 시 코드 블록으로 남을 수 있음 |
| KaTeX 수식 | 수식 또는 계산 설명 | 결과와 단위 확인 | 업무 기준 수치는 재검산 |
| 인라인 이미지 | 생성 이미지, 첨부 이미지, 도구 결과 | 확대 또는 다운로드 | 공유 전 이미지 안의 민감정보 확인 |
| 계획 카드 | todo 또는 단계형 작업 | 진행 중/완료 항목 확인 | 오래 멈추면 trace와 tool state 확인 |
| 검색 결과 카드 | 검색 도구 호출 | 제목, URL, snippet 확인 | 외부 링크 신뢰도 별도 판단 |
| 파일 도구 카드 | 파일 읽기/쓰기/수정 | 대상 파일명과 결과 확인 | 경로, 권한, overwrite 여부 확인 |
| 일반 도구 카드 | 전용 UI가 없는 도구 | 입력, 상태, 결과, 오류 확인 | 모델 답변보다 카드 오류가 더 정확할 수 있음 |
| 승인 카드 | tool/prompt/recommendation approval | 승인, 거절, 수정 입력 | 만료되면 resume이 불가능할 수 있음 |
| 사용자 입력 카드 | 명확화 질문, 선택지, HITL | 자유 입력 또는 선택 | 답변에 따라 이후 실행이 달라짐 |
| 메모리 카드 | memory proposal 또는 save event | 저장 전 내용 검토 | 민감정보는 승인하지 않음 |
| 서브에이전트 카드 | 다른 agent/task에 위임 | 위임 대상과 결과 확인 | 실패 시 parent trace도 확인 |
| Deep research 요약 | 조사형 작업 결과 | source와 요약 구조 확인 | 외부 근거와 최신성 확인 |
| Builder 카드 | 에이전트 생성 빌더 | 추천 승인, 이미지 선택, draft config 확인 | 확정 전 설정을 다시 검토 |
| Artifact 카드 | 생성 파일 기록 | preview, favorite, download | 파일 상태가 `ready`인지 확인 |

## Builder 단계 카드

| 카드 | 목적 | 확인할 것 |
| --- | --- | --- |
| Phase timeline | 대화형 생성 흐름의 현재 단계 표시 | 어느 단계가 완료/진행 중인지 |
| 추천 도구 검토 | 에이전트에 붙일 도구 후보 제안 | 필요 없는 도구, 권한 범위 |
| 시스템 프롬프트 승인 | 생성된 system prompt 검토 | 역할, 금지사항, 출력 형식 |
| 이미지 선택 | 후보 이미지 중 선택 | 브랜드 톤과 안전성 |
| 이미지 승인 | 선택된 이미지 최종 확인 | 공개 가능 여부 |
| Draft 설정 미리보기 | 모델, 도구, 스킬, 스케줄, middleware 구성 확인 | 빠진 설정과 과한 권한 |
| 최종 설정 승인 | 에이전트 생성 전 마지막 승인 | 이름, 설명, 도구, prompt |

## 상태 읽는 법

| 상태 | 의미 | 다음 행동 |
| --- | --- | --- |
| 스트리밍 중 | 메시지와 카드가 아직 완성되지 않음 | 완료 이벤트까지 기다림 |
| 대기 중 | 승인, 사용자 입력, 도구 응답을 기다림 | 카드 안의 버튼이나 입력을 처리 |
| 완료 | 결과가 응답에 반영됨 | 요약, 파일, trace, usage 확인 |
| 실패 | 도구, credential, model, MCP, storage 중 일부 실패 | 오류 카드와 trace를 먼저 확인 |
| 만료 | 승인 또는 입력 window가 지나감 | 새 메시지로 다시 실행하거나 resume 가능 여부 확인 |

## 승인과 메모리 확인 기준

| 카드 | 확인 기준 |
| --- | --- |
| 승인 카드 | 대상 도구, 입력 JSON, 외부 호출, 파일 변경, 공유, 자격증명 사용 |
| 사용자 입력 카드 | 질문의 의도, 선택지 의미, 자유 입력이 이후 실행에 주는 영향 |
| 메모리 카드 | 사실성, 민감도, 저장 범위, 짧고 수정 가능한 표현 |

## 파일과 artifact 카드

Artifact 카드는 대화 중 생성된 파일의 가장 가까운 진입점입니다. 파일 형식별 preview provider와 fallback은 [아티팩트 뷰어 매트릭스](/hancom/moldy/ko/artifact-viewers)에서, 파일 라이브러리 흐름은 [파일과 아티팩트](/hancom/moldy/ko/files-artifacts)에서 확인하세요.

| 카드에서 확인할 것 | 이유 |
| --- | --- |
| 파일 이름과 확장자 | preview provider가 어떤 기준으로 선택되는지 확인 |
| status | `ready`가 아니면 download/preview가 실패할 수 있음 |
| source tool call | 어떤 도구가 파일을 만들었는지 추적 |
| version | 같은 파일이 여러 번 갱신되었는지 확인 |
| share 상태 | 공개 링크에 포함될 수 있는지 확인 |
