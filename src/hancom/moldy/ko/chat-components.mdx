Moldy 채팅 화면은 텍스트 답변뿐 아니라 Markdown, 코드, 다이어그램, 수식, 이미지, 검색 결과, 도구 호출, 승인 요청, 사용자 입력, 메모리 제안, 서브에이전트, Builder 단계, 생성 파일을 전용 컴포넌트로 표시합니다.

이 페이지는 채팅에서 보이는 UI를 빠르게 해석하기 위한 갤러리입니다. 실제 대화 흐름은 [에이전트와 채팅하기](/hancom/moldy/ko/chat-with-agent)를, 스트리밍 API는 [채팅 스트리밍 API](/hancom/moldy/ko/api-chat-streaming)를 참고하세요.

아래 갤러리는 Moldy 프론트엔드의 실제 채팅 컴포넌트와 스타일을 렌더링해 캡처한 화면입니다. 합성 목업이 아니라 제품의 `ToolUI`, Markdown renderer, Builder card 컴포넌트를 기준으로 합니다.

## 컴포넌트 빠른 표

| 컴포넌트 | 나타나는 상황 | 사용자가 할 일 | 실패 또는 주의 |
| --- | --- | --- | --- |
| Markdown 본문 | 일반 답변, 설명, 요약 | heading, 표, 링크를 읽고 다음 요청 작성 | 스트리밍 중 표가 임시로 깨질 수 있음 |
| 코드 블록 | 코드, 로그, 설정 예시 | 언어 라벨과 내용을 확인 | 실행 가능한 코드인지 별도 검증 필요 |
| Mermaid | `mermaid` 코드 블록 | 다이어그램 구조 확인 | 문법 오류 시 코드 블록으로 남을 수 있음 |
| KaTeX 수식 | 수식 또는 계산 설명 | 결과와 단위 확인 | 모델 계산은 업무 기준으로 재검산 |
| 이미지 | 생성 이미지, 첨부 이미지, 도구 결과 | 확대 또는 다운로드 | 공유 전 이미지 내 민감정보 확인 |
| 검색 결과 카드 | 검색 도구 호출 | 제목, URL, snippet 확인 | 외부 링크 신뢰도 별도 판단 |
| 파일 도구 카드 | 파일 읽기/쓰기/수정 | 대상 파일명과 결과 확인 | 경로, 권한, overwrite 여부 확인 |
| 일반 도구 카드 | 전용 UI가 없는 도구 | 입력, 상태, 결과, 오류 확인 | 모델 답변보다 카드 오류가 더 정확할 수 있음 |
| 계획 카드 | todo 또는 단계형 작업 | 진행 중/완료 항목 확인 | 오래 멈추면 trace와 tool state 확인 |
| 승인 카드 | tool/prompt/recommendation approval | 승인, 거절, 수정 입력 | 만료되면 resume이 불가능할 수 있음 |
| 사용자 입력 카드 | 명확화 질문, 선택지, HITL | 자유 입력 또는 선택 | 잘못 답하면 이어지는 실행도 달라짐 |
| 메모리 카드 | memory proposal 또는 save event | 저장 전 내용 검토 | 민감정보는 승인하지 않음 |
| 서브에이전트 카드 | 다른 agent/task에 위임 | 위임 대상과 결과 확인 | 실패 시 원 agent의 trace도 함께 확인 |
| Builder 카드 | 에이전트 생성 빌더 | 추천 승인, 이미지 선택, draft config 확인 | 확정 전 설정을 다시 검토 |
| Artifact 카드 | 생성 파일 기록 | preview, favorite, download | 파일 상태가 `ready`인지 확인 |

## 상태 읽는 법

| 상태 | 의미 | 다음 행동 |
| --- | --- | --- |
| 스트리밍 중 | 메시지와 카드가 아직 완성되지 않음 | 완료 이벤트까지 기다림 |
| 대기 중 | 승인, 사용자 입력, 도구 응답을 기다림 | 카드 안의 버튼이나 입력을 처리 |
| 완료 | 결과가 응답에 반영됨 | 요약, 파일, trace, usage 확인 |
| 실패 | 도구, credential, model, MCP, storage 중 일부 실패 | 오류 카드와 trace를 먼저 확인 |
| 만료 | 승인 또는 입력 window가 지나감 | 새 메시지로 다시 실행하거나 resume 가능 여부 확인 |

## 승인 카드

승인 카드는 사용자가 실행을 계속할지 결정해야 하는 지점입니다. Moldy는 승인 대상에 따라 tool approval, prompt approval, recommendation approval처럼 다른 맥락을 표시할 수 있습니다.

1. 카드 제목과 대상 도구를 확인합니다.
2. 입력 JSON 또는 요약을 읽습니다.
3. 위험한 외부 호출, 파일 변경, 공유, 자격증명 사용이 있는지 확인합니다.
4. 그대로 진행할 수 있으면 승인합니다.
5. 입력을 바꿔야 하면 수정 JSON을 사용합니다.
6. 불확실하면 거절하고 에이전트 지침이나 도구 설정을 고칩니다.

## 메모리 카드

메모리 카드는 사용자 또는 에이전트 장기 메모리에 저장할 후보를 보여줍니다. 메모리는 다음 대화 품질을 높일 수 있지만, 민감정보를 오래 남길 위험도 있습니다.

| 확인 항목 | 기준 |
| --- | --- |
| 사실성 | 사용자가 실제로 장기적으로 유지하고 싶은 정보인지 |
| 민감도 | 비밀번호, token, 개인식별정보, 고객 정보가 아닌지 |
| 범위 | 모든 에이전트에 필요한지, 특정 에이전트에만 필요한지 |
| 표현 | 짧고 수정 가능한 문장인지 |

## 파일과 artifact 카드

Artifact 카드는 대화 중 생성된 파일의 가장 가까운 진입점입니다. 같은 파일은 [파일과 아티팩트](/hancom/moldy/ko/files-artifacts)와 [아티팩트 뷰어 매트릭스](/hancom/moldy/ko/artifact-viewers)에서도 확인할 수 있습니다.

| 카드에서 확인할 것 | 이유 |
| --- | --- |
| 파일 이름과 확장자 | preview provider가 어떤 기준으로 선택되는지 확인 |
| status | `ready`가 아니면 download/preview가 실패할 수 있음 |
| source tool call | 어떤 도구가 파일을 만들었는지 추적 |
| version | 같은 파일이 여러 번 갱신되었는지 확인 |
| share 상태 | 공개 링크에 포함될 수 있는지 확인 |
