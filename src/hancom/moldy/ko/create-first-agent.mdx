description: 대화형 빌더, 직접 만들기, 템플릿으로 Moldy 에이전트를 생성하고 검증하는 방법입니다.
---

Moldy는 같은 결과, 즉 채팅에서 실행할 수 있는 저장된 에이전트를 만들기 위해 세 가지 생성 방식을 제공합니다. 대화로 만들기, 직접 만들기, 템플릿으로 만들기 중 하나를 선택한 뒤 에이전트 화면에서 결과를 검증합니다.

실제 라우트는 `/agents/new`, `/agents/new/conversational`, `/agents/new/manual`, `/agents/new/template`로 나뉩니다. 이 문서는 모든 팀에 하나의 영구 워크플로를 권장하기보다, 각 생성 방식 이후 무엇을 확인해야 하는지에 초점을 둡니다.

<img className="moldy-screenshot" src="/images/hancom/moldy/ko/agent-create.png" alt="Moldy 새 에이전트 생성 시작 화면" />

## 생성 방식 선택

| 방식 | 적합한 상황 | 생성 후 확인할 것 |
| --- | --- | --- |
| 대화로 만들기 | 목표를 자연어로 설명하고 구성을 제안받고 싶을 때 | 제안된 지침, 도구, 모델 |
| 직접 만들기 | 필요한 설정을 이미 알고 있을 때 | 모델, 도구, 스킬, 미들웨어, 스케줄 |
| 템플릿으로 만들기 | 준비된 업무 패턴에서 시작할 때 | 템플릿 지침과 필요한 자격증명 |

## 대화로 만들기

<img className="moldy-screenshot" src="/images/hancom/moldy/ko/agent-create-conversational.png" alt="자연어로 에이전트를 만드는 Moldy 대화형 빌더 화면" />

대화형 빌더는 사용자의 설명을 기반으로 에이전트 초안을 만듭니다. 백엔드에는 builder session, builder messages, resume, confirm API가 있어 단계적으로 구성을 검토하는 흐름을 지원합니다.

<img className="moldy-screenshot" src="/images/hancom/moldy/ko/agent-create-conversational-progress.png" alt="8단계 phase timeline과 검토 카드가 표시된 Moldy 대화형 빌더 진행 화면" />

빌더는 8단계 phase로 진행됩니다. 프로젝트 초기화와 의도 분석을 시작으로 도구 추천, 미들웨어 추천, 시스템 프롬프트, 이미지 선택, 설정 저장, 최종 빌드 확인까지 이어집니다. 중간 단계에서 검토 카드가 나오면 승인하거나 수정 의견을 입력해 다음 단계로 넘깁니다.

좋은 요청에는 다음 요소를 넣습니다.

- 에이전트가 해결할 업무
- 사용할 입력 데이터 또는 외부 서비스
- 결과 형식
- 자동 실행이 필요한 경우 주기
- 금지해야 할 행동이나 확인해야 할 조건

```txt
매주 월요일 오전에 지난주 사용량을 요약하고,
비용이 많이 든 에이전트 3개와 원인을 표로 정리해줘.
```

## 직접 만들기

<img className="moldy-screenshot" src="/images/hancom/moldy/ko/agent-create-manual.png" alt="프롬프트, 모델, 도구와 스킬을 직접 설정하는 Moldy 에이전트 편집 화면" />

직접 만들기는 생성 시점부터 에이전트 설정 화면과 거의 같은 필드를 제공합니다.

| 필드 | 설명 |
| --- | --- |
| 이름/설명 | 대시보드, 사이드바, 공유 화면에서 사용 |
| 지침 | 에이전트의 역할과 응답 규칙 |
| 모델 | 기본 모델과 모델 파라미터 |
| 도구/MCP 도구 | 실행 가능한 액션 |
| 스킬 | 재사용 지식과 파일 |
| 서브에이전트 | 보조 실행 단위 |
| 미들웨어 | 런타임 확장 설정 |
| 오프너 | 채팅 시작 추천 질문 |
| 스케줄 | 자동 실행 trigger |

직접 만들기에서 저장 버튼이 활성화되려면 이름과 모델이 필요합니다.

## 템플릿으로 만들기

<img className="moldy-screenshot" src="/images/hancom/moldy/ko/agent-templates.png" alt="Moldy 템플릿 목록 화면" />

템플릿은 기본 지침과 도구 구성을 제공하는 시작점입니다. 생성 후에는 일반 에이전트처럼 설정을 수정할 수 있습니다.

템플릿을 사용할 때 확인할 점은 다음과 같습니다.

- 템플릿이 요구하는 도구가 현재 사용자에게 사용 가능한지
- 필요한 사용자 자격증명이 등록되어 있는지
- 템플릿 지침이 실제 업무 환경에 맞는지
- 공유 또는 마켓플레이스 게시 전에 이름/설명/아이콘이 적절한지

## 생성 후 첫 검증

1. 에이전트 화면을 엽니다.
2. 짧은 테스트 메시지를 보냅니다.
3. 도구 호출이 있다면 결과 패널을 확인합니다.
4. 기대와 다르면 [에이전트 설정 관리](/hancom/moldy/ko/agent-settings)에서 지침, 모델, 도구를 조정합니다.
5. 사용량 화면에서 요청과 비용 기록을 확인합니다.

첫 검증은 에이전트가 기대한 형식으로 응답하고, 의도한 도구만 사용하며, 사용량이 기록될 때 완료된 것으로 봅니다. 빌더가 만든 초안이라면 다른 사용자에게 공유하기 전에 첫 검증을 설정 검토 단계로 활용하세요.

<Tip>
처음에는 도구 호출이 필요 없는 작은 요청으로 시작하세요. 기본 응답 형식이 맞은 뒤 도구와 스케줄을 추가하면 문제를 더 쉽게 분리할 수 있습니다.
</Tip>
