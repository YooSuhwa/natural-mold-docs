Moldy 문제 해결은 UI, 네트워크 응답, 런타임 trace, 관련 설정을 함께 수집할 때 가장 빠릅니다. 화면 메시지만으로는 인증, 권한, 자격증명 범위, 모델 설정, 도구 런타임 문제 중 무엇인지 구분하기 어렵습니다.

Moldy는 리소스 소유권 보호를 위해 존재하지 않는 리소스와 권한 없는 리소스를 같은 404 계열로 처리할 수 있습니다. 리소스가 보이지 않을 때는 삭제되었다고 보기 전에 소유권과 `super_user` 권한을 확인하세요.

상태 코드, 카드 상태, MCP, artifact, Agent API 오류를 더 세부적으로 분류하려면 [오류 메시지 사전](/hancom/moldy/ko/error-reference)을 참고하세요.

## 로그인 후 다시 로그인 화면으로 이동합니다

- 프론트엔드의 API base URL이 실제 백엔드 포트를 가리키는지 확인합니다.
- 백엔드 CORS 허용 origin에 프론트엔드 origin이 포함되어 있는지 확인합니다.
- 브라우저 쿠키가 차단되어 있지 않은지 확인합니다.
- `/api/auth/me`가 현재 사용자를 반환하는지 확인합니다.
- 세션 만료 후 mutation이 실패한다면 다시 로그인한 뒤 시도합니다.

로그인은 한 번 성공했지만 mutation에서 실패한다면 계정 정보를 바꾸기 전에 CSRF 처리를 먼저 확인하세요.

## 상태 변경 요청이 실패합니다

- `POST`, `PATCH`, `DELETE` 요청에 CSRF 값이 포함되는지 확인합니다.
- 브라우저 앱 밖에서 API를 호출한다면 로그인 쿠키와 CSRF 헤더를 함께 보내야 합니다.
- 운영자 endpoint라면 계정에 `super_user` 권한이 있는지 확인합니다.

## Builder 또는 Assistant가 동작하지 않습니다

- 운영자 계정으로 **시스템 자격증명**을 확인합니다.
- **System LLM 설정**에서 텍스트 기본, 텍스트 fallback, 이미지 슬롯이 모두 설정되었는지 확인합니다.
- 모델 화면에서 선택한 모델 상태 확인을 실행합니다.
- 시스템 자격증명 definition이 LLM provider에 맞는지 확인합니다.

## 채팅 응답이 오지 않거나 중간에 끊깁니다

- 에이전트에 기본 모델이 연결되어 있는지 확인합니다.
- 사용자 LLM 자격증명이 준비되어 있는지 확인합니다.
- 스트림 재연결이 필요한 경우 run id 기반 `/stream` 경로를 확인합니다.
- 도구 interrupt 이후라면 resume 버튼 또는 resume API 흐름을 확인합니다.
- trace 화면에서 실패 단계와 오류 메시지를 확인합니다.

## 도구 실행이 실패합니다

- 도구가 에이전트에 연결되어 있는지 확인합니다.
- 도구에 필요한 사용자 자격증명이 등록되어 있는지 확인합니다.
- 모델이 tool calling을 지원하는지 확인합니다.
- 도구 결과 패널의 오류 메시지를 먼저 읽습니다.

## MCP 서버 probe 또는 도구 발견이 실패합니다

- transport가 `streamable_http`/`sse`이면 URL이 필요합니다.
- transport가 `stdio`이면 command가 필요합니다.
- header/env var의 credential template이 올바른지 확인합니다.
- 선택한 credential이 현재 사용자 소유인지 확인합니다.
- 저장 후 실패했다면 **상태 확인**과 **도구 발견**을 다시 실행합니다.

## 스케줄이 실행되지 않습니다

- trigger status가 `active`인지 확인합니다.
- `next_run_at`, timezone, cron expression, end_at을 확인합니다.
- `max_runs`에 도달하지 않았는지 확인합니다.
- 실패가 누적되어 자동 pause 조건에 걸리지 않았는지 확인합니다.
- 결과 대화가 target conversation 또는 schedule conversation에 연결되어 있는지 확인합니다.

## 마켓플레이스 설치가 `needs_setup` 상태입니다

- 항목 최신 버전의 credential requirements를 확인합니다.
- 필수 요구사항에 사용자 자격증명을 바인딩합니다.
- credential definition이 요구사항과 일치하는지 확인합니다.
- 설치 업데이트 전략을 선택할 때 현재 리소스를 덮어쓸지 새 복사본을 만들지 결정합니다.

## 공유 링크가 404를 반환합니다

- share token이 회수되었는지 확인합니다.
- 대화 소유자가 활성 공유 링크를 다시 생성했는지 확인합니다.
- 공개 endpoint는 존재하지 않는 token과 회수된 token 모두 404로 응답할 수 있습니다.

## 문서 캡처를 다시 만들고 싶습니다

```bash
pnpm run capture
pnpm run check:sensitive
```

캡처 스크립트는 `.env`의 계정으로 로그인하고, 저장 전에 이메일, 사용자명, 내부 URL, 실제 비용 숫자를 예시 값으로 치환합니다.

문서 캡처 재생성은 문서 검증의 일부입니다. 새 스크린샷을 저장한 뒤에는 민감정보가 보이는 화면을 게시 전에 직접 확인하세요.
