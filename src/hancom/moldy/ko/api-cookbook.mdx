API Cookbook은 OpenAPI 스키마를 읽기 전에 자주 필요한 호출 흐름을 빠르게 시도하는 문서입니다. 브라우저 앱용 `/api/*` endpoint는 세션 쿠키와 CSRF가 필요하고, 외부 런타임용 `/v1/*` endpoint는 Agent API key가 필요합니다.

정확한 endpoint 목록과 schema는 [API Reference](/hancom/moldy/ko/api-reference)와 [백엔드 API 전체 목록](/hancom/moldy/ko/api-endpoints)을 기준으로 확인하세요.

<Warning>
예제에 실제 비밀번호, 쿠키, CSRF token, Agent API key를 넣어 문서나 채팅에 공유하지 마세요.
</Warning>

## 공통 변수

```bash
export MOLDY_BASE_URL="https://your-moldy-host.example"
export MOLDY_EMAIL="user@example.com"
export MOLDY_PASSWORD="replace-with-your-password"
```

로컬 개발에서는 `MOLDY_BASE_URL`을 실제 백엔드 origin으로 바꿉니다. 프론트엔드 origin이 아니라 API 서버 origin을 사용해야 합니다.

## 로그인하고 CSRF 저장

`POST /api/auth/login`은 HttpOnly session cookie를 설정하고 응답 body에 `csrf_token`을 반환합니다.

```bash
curl -sS -c cookies.txt \
  -H "Content-Type: application/json" \
  -X POST "$MOLDY_BASE_URL/api/auth/login" \
  -d "{\"email\":\"$MOLDY_EMAIL\",\"password\":\"$MOLDY_PASSWORD\"}" \
  | tee login.json

export CSRF_TOKEN="$(node -e "console.log(require('./login.json').csrf_token)")"
```

상태 변경 요청에는 cookie jar와 `X-CSRF-Token`을 함께 보냅니다.

```bash
curl -sS -b cookies.txt \
  -H "X-CSRF-Token: $CSRF_TOKEN" \
  "$MOLDY_BASE_URL/api/auth/me"
```

## 대화 만들고 메시지 보내기

대화 생성 payload는 `ConversationCreate` schema를 따르며 `title`은 선택입니다.

```bash
export AGENT_ID="00000000-0000-0000-0000-000000000000"

curl -sS -b cookies.txt \
  -H "Content-Type: application/json" \
  -H "X-CSRF-Token: $CSRF_TOKEN" \
  -X POST "$MOLDY_BASE_URL/api/agents/$AGENT_ID/conversations" \
  -d '{"title":"API cookbook test"}' \
  | tee conversation.json

export CONVERSATION_ID="$(node -e "const v=require('./conversation.json'); console.log(v.id || v.conversation_id)")"
```

메시지 전송 payload는 `MessageCreate` schema를 따릅니다. 응답은 SSE stream일 수 있으므로 터미널에서는 chunk가 이어서 출력됩니다.

```bash
curl -N -b cookies.txt \
  -H "Content-Type: application/json" \
  -H "X-CSRF-Token: $CSRF_TOKEN" \
  -X POST "$MOLDY_BASE_URL/api/conversations/$CONVERSATION_ID/messages" \
  -d '{"content":"이 에이전트가 할 수 있는 일을 세 문장으로 요약해줘."}'
```

## 스트림 재연결과 resume

스트림이 끊기면 client가 보관한 run id로 재연결 경로를 확인합니다.

```bash
export RUN_ID="run-id-from-client-or-response"

curl -N -b cookies.txt \
  "$MOLDY_BASE_URL/api/conversations/$CONVERSATION_ID/stream?run_id=$RUN_ID"
```

승인 또는 interrupt 이후 resume이 필요한 경우 `ResumeRequest`는 `decisions` 배열을 보냅니다. decision 구조는 실행 카드가 요구하는 값에 맞춰야 하므로 UI 카드와 OpenAPI schema를 함께 확인하세요.

```bash
curl -N -b cookies.txt \
  -H "Content-Type: application/json" \
  -H "X-CSRF-Token: $CSRF_TOKEN" \
  -X POST "$MOLDY_BASE_URL/api/conversations/$CONVERSATION_ID/messages/resume" \
  -d '{"decisions":[]}'
```

## 파일과 artifact 조회

```bash
curl -sS -b cookies.txt \
  "$MOLDY_BASE_URL/api/artifacts?limit=20"

export ARTIFACT_ID="00000000-0000-0000-0000-000000000000"

curl -L -b cookies.txt \
  "$MOLDY_BASE_URL/api/artifacts/$ARTIFACT_ID/download" \
  -o artifact-download.bin
```

대화에 속한 파일만 보려면 conversation endpoint를 사용합니다.

```bash
curl -sS -b cookies.txt \
  "$MOLDY_BASE_URL/api/conversations/$CONVERSATION_ID/artifacts"
```

## 공유 링크 만들기와 회수

공유 링크는 로그인 없는 read-only snapshot을 만들 수 있으므로 공개 전 내용을 검토합니다.

```bash
curl -sS -b cookies.txt \
  -H "X-CSRF-Token: $CSRF_TOKEN" \
  -X POST "$MOLDY_BASE_URL/api/conversations/$CONVERSATION_ID/share"

curl -sS -b cookies.txt \
  -H "X-CSRF-Token: $CSRF_TOKEN" \
  -X DELETE "$MOLDY_BASE_URL/api/conversations/$CONVERSATION_ID/share"
```

## Agent API 런타임 호출

Agent API는 브라우저 cookie가 아니라 server-side API key를 사용합니다. 키는 **Settings > Agent API**에서 만들고 안전한 secret storage에 저장합니다.

```bash
export MOLDY_AGENT_API_KEY="moldy-agent-api-key"
export DEPLOYMENT_PUBLIC_ID="agent-public-id"

curl -sS \
  -H "Authorization: Bearer $MOLDY_AGENT_API_KEY" \
  -H "Content-Type: application/json" \
  -X POST "$MOLDY_BASE_URL/v1/runs/wait" \
  -d '{
    "agent_id": "'"$DEPLOYMENT_PUBLIC_ID"'",
    "input": {
      "messages": [
        {"role": "user", "content": "오늘 해야 할 일을 짧게 정리해줘."}
      ]
    },
    "metadata": {"source": "api-cookbook"}
  }'
```

stream scope가 있으면 `/v1/runs/stream`을 사용할 수 있습니다.

```bash
curl -N \
  -H "Authorization: Bearer $MOLDY_AGENT_API_KEY" \
  -H "Content-Type: application/json" \
  -X POST "$MOLDY_BASE_URL/v1/runs/stream" \
  -d '{
    "agent_id": "'"$DEPLOYMENT_PUBLIC_ID"'",
    "input": {
      "messages": [
        {"role": "user", "content": "streaming으로 진행 상황을 보여줘."}
      ]
    },
    "stream_mode": ["updates"]
  }'
```

## 실패할 때 빠른 분류

| 증상 | 먼저 볼 것 |
| --- | --- |
| `401` | cookie 만료, Agent API key 누락, revoked key |
| `403` | CSRF mismatch, `super_user` 권한 부족, scope 부족 |
| `404` | 다른 사용자 소유 리소스, 회수된 share token, 잘못된 public ID |
| stream 중단 | run id, reconnect path, trace |
| artifact download 실패 | artifact status, owner permission, storage object |

더 자세한 분류는 [오류 메시지 사전](/hancom/moldy/ko/error-reference)을 참고하세요.
