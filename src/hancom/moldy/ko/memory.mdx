메모리는 사용자가 승인한 사실을 Moldy 에이전트가 대화 간에 다시 사용할 수 있게 해 줍니다. 설정은 두 단계로 나뉩니다. Settings의 사용자 전역 메모리 정책과 에이전트 설정의 선택적 에이전트별 override입니다.

## 언제 쓰는가

메모리는 안정적인 선호, 반복되는 사실, 에이전트 전용 맥락을 기억해야 할 때 사용합니다. 일회성 지시, secret, API key, private document, 빨리 만료되어야 하는 사실은 메모리에 저장하지 마세요.

| 범위 | 의미 | 예시 |
| --- | --- | --- |
| 사용자 | 여러 에이전트에서 사용자 선호로 사용 | 선호 언어, 보고서 형식, 시간대 선호 |
| 에이전트 | 특정 에이전트에서만 사용 | 지원 에이전트의 고객 요약 필드 |

## 전역 메모리 정책

사용자 정책은 `/api/me/memory-settings`로 저장됩니다.

| 설정 | 지원 값 | 효과 |
| --- | --- | --- |
| 메모리 활성화 | on/off | 메모리 읽기와 쓰기를 켜거나 끔 |
| 응답에 메모리 사용 | on/off | 활성 메모리를 런타임 프롬프트에 포함 |
| 대화 중 저장 | `off`, `ask`, `auto` | 대화가 메모리를 저장할 수 있는지 결정 |
| 저장 범위 | `user`, `agent`, `both` | 저장 가능한 메모리 범위 제한 |
| 자동 실행 저장 | `off`, `auto` | 스케줄 실행 중 메모리 저장 허용 여부 |

운영 기본값은 사용자가 저장될 내용을 이해할 때까지 쓰기를 `ask`로 두는 것이 안전합니다. `auto`는 에이전트의 도메인과 privacy boundary가 명확할 때만 사용하세요.

## 기록 추가, 수정, 삭제

1. **Settings > Memory**를 엽니다.
2. **User** 또는 **Agent** 범위를 선택합니다.
3. 집중된 메모리 내용을 입력합니다.
4. 나중에 검토할 수 있도록 선택적으로 저장 이유를 입력합니다.
5. 저장한 뒤 기록된 메모리 목록에서 확인합니다.

저장된 메모리에는 status, timestamp, 가능한 경우 source conversation/run field, user profile memory 또는 agent memory 같은 store path가 포함됩니다. 삭제된 메모리는 런타임에서 사용되지 않습니다.

## 에이전트 override

에이전트 설정은 사용자 정책을 다음 값으로 override할 수 있습니다.

| Override | 값 | 사용할 때 |
| --- | --- | --- |
| Memory policy | `inherit`, `off`, `ask`, `auto` | 특정 에이전트만 더 엄격하거나 더 허용적인 쓰기 정책이 필요할 때 |
| Scope policy | `inherit`, `agent_only`, `user_and_agent` | 특정 에이전트가 사용자 공통 메모리를 쓰지 않아야 할 때 |
| Trigger policy | `inherit`, `off`, `auto` | 스케줄 실행의 메모리 동작을 따로 정해야 할 때 |

에이전트 override는 민감한 에이전트에 유용합니다. 임시 정보나 private content를 다루는 에이전트라면 쓰기를 `off`로 두거나 확인을 요구하세요.

## 메모리 제안

쓰기 정책이 `ask`이면 런타임은 바로 저장하지 않고 제안을 만들 수 있습니다. 제안은 승인, 수정 후 승인, 거절할 수 있습니다. 제안 내용은 직접 저장한 메모리와 같은 길이 제한과 non-empty validation을 적용받습니다.

<Warning>
메모리는 사용자가 제어하는 참고 정보이지 secret 저장소가 아닙니다. API key, password, private token, confidential document text를 메모리에 저장하지 마세요.
</Warning>

## 실패 시 확인

| 증상 | 확인할 항목 |
| --- | --- |
| 에이전트가 메모리를 반영하지 않음 | 메모리 활성화, 읽기 활성화, 기록 범위가 에이전트와 맞는지 확인 |
| 메모리가 저장되지 않음 | write policy, allowed scope, 승인 대기 중인 proposal 여부 확인 |
| 스케줄 실행에서 저장되지 않음 | 사용자와 에이전트의 trigger write policy 확인 |
| 메모리 수정 실패 | 내용이 비어 있지 않고 길이 제한 안에 있는지 확인 |

## 관련 API

- `/api/me/memory-settings`
- `/api/agents/{agent_id}/memory-settings`
- `/api/memories`
- `/api/memory-proposals`
