Moldy 보안은 사용자 소유 리소스, 운영자 전용 설정, 공개 공유 링크, Agent API key, 장기 메모리, 생성 파일을 구분해서 다루는 것이 핵심입니다. 이 문서는 기능별 데이터가 어디에 남고, 어떤 내용을 공유 전에 확인해야 하는지 설명합니다.

접근 제어의 기본 모델은 [접근 권한과 감사](/hancom/moldy/ko/access-oversight)를 참고하세요.

## 데이터 표면

| 표면 | 저장 또는 노출될 수 있는 정보 | 기본 원칙 |
| --- | --- | --- |
| 대화 메시지 | 사용자 입력, 모델 응답, 도구 결과 | 민감정보를 넣기 전에 agent와 share 범위 확인 |
| 도구 호출 | 입력 JSON, 외부 API 결과, 오류 | credential 값이 결과에 노출되지 않게 확인 |
| 메모리 | 장기 저장될 사용자/에이전트 정보 | 저장 전 proposal을 검토 |
| 생성 파일 | 문서, 표, 이미지, 코드, 로그 | preview와 download 전에 내용 확인 |
| 공유 링크 | read-only 대화 snapshot과 공유 파일 | 공개 전 메시지와 파일 모두 검토 |
| Agent API | server-side key, deployment, run metadata | key scope와 deployment 제한 설정 |
| 감사 로그 | credential 변경, Agent API key 변경, 운영자 작업 | secret 값 없이 변경 이력만 추적 |

## 자격증명

자격증명 응답은 복호화된 secret 값을 반환하지 않습니다. 사용자 자격증명은 사용자 소유 범위에서, 시스템 자격증명은 운영자 범위에서 관리합니다.

| 해야 할 일 | 이유 |
| --- | --- |
| credential 이름과 설명은 secret 없이 작성 | 화면 캡처와 audit log에 안전하게 표시 |
| 테스트 결과에 secret이 포함되지 않는지 확인 | 도구 결과나 오류 카드로 새어 나가는 것을 방지 |
| 사용자 credential과 시스템 credential을 혼동하지 않음 | Builder/System LLM과 사용자 에이전트 실행을 분리 |
| credential audit log 확인 | 누가 언제 변경했는지 추적 |

## 메모리

메모리는 다음 대화 품질을 높이지만 오래 남는 정보입니다. 메모리 proposal은 저장 전 검토해야 합니다.

| 저장해도 좋은 예 | 저장하면 안 되는 예 |
| --- | --- |
| 선호하는 응답 형식 | 비밀번호, API key, token |
| 반복 업무의 공개적인 맥락 | 주민등록번호, 고객 식별정보 |
| 장기 프로젝트 이름과 역할 | 일회성 비밀 요청 |
| 사용자가 명시적으로 기억하길 원하는 설정 | 규정상 보존하면 안 되는 데이터 |

## 생성 파일과 artifact

생성 파일은 대화보다 더 쉽게 다운로드되고 공유될 수 있습니다. 특히 DOCX, PPTX, XLSX, PDF, HTML, 이미지에는 메시지 본문보다 더 많은 정보가 포함될 수 있습니다.

공유 전 확인:

1. 파일 preview를 엽니다.
2. 표, 이미지, header/footer, speaker note, hidden sheet처럼 눈에 덜 띄는 영역을 확인합니다.
3. source tool call과 run ID를 확인합니다.
4. public share에 포함될 파일인지 확인합니다.
5. 필요하면 share를 만들기 전에 파일을 삭제하거나 대화를 분리합니다.

## 공유 링크

공유 링크는 로그인 없는 공개 조회입니다. 링크를 가진 사람은 공유된 snapshot과 포함된 artifact를 볼 수 있습니다.

| 상황 | 권장 행동 |
| --- | --- |
| 내부 검토용 공유 | 민감 파일 제거 후 짧은 대화만 공유 |
| 외부 고객 공유 | 메시지, 도구 결과, artifact preview를 모두 확인 |
| 잘못 공유함 | 즉시 share revoke |
| 공유 링크가 404 | token 회수 또는 알 수 없는 token으로 처리될 수 있음 |

## Agent API key

Agent API key는 브라우저용 키가 아닙니다. 서버 secret storage에만 저장하고 public client code, screenshot, 문서 예제, 채팅 transcript에 넣지 않습니다.

| 설정 | 권장 |
| --- | --- |
| scope | 필요한 scope만 부여 |
| deployment | 가능하면 selected deployment로 제한 |
| expiry | 외부 통합 목적에 맞게 만료일 설정 |
| rotation | 노출 의심 시 즉시 revoke 후 재발급 |
| logging | key name, prefix/last four, deployment public ID, run ID만 기록 |

## 문서와 스크린샷

문서 작업자는 다음 값을 문서와 이미지에 남기지 않습니다.

- 실제 cookie, CSRF token, API key, credential secret.
- 실제 이메일, 사용자명, 고객명, 내부 URL.
- 생성 파일 안의 민감 본문.
- 비공개 모델 base URL 또는 내부 시스템 경로.

스크린샷을 다시 캡처했다면 `pnpm run check:sensitive`를 실행하고, 새 화면을 직접 확인하세요.
