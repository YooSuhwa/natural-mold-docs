Recipes are not a feature-by-feature reference. They show which Moldy screens to open, in what order, to finish a real task. This page connects agent creation, tools, MCP servers, skills, chat validation, generated files, schedules, and Agent API deployment into practical workflows.

When a new feature is documented, add or update a recipe here if that feature changes what users can accomplish.

## Choose a recipe

| Goal | Start here | Preparation | Result |
| --- | --- | --- | --- |
| Document analysis agent | **New Agent** or **Agent Templates** | Model, user credential, document tools or skills | Summaries, comparisons, checklists in chat |
| Recurring report automation | Agent settings and **Schedules** | Stable prompt, model, required tools | Scheduled runs and result conversations/files |
| External tool connection | **Capabilities > MCP Servers** | MCP transport, URL or command, credential | MCP tools available in agent settings |
| Generated file review and sharing | Chat artifact card or **Settings > Files** | File-generating tool or skill, review policy | Preview, favorite, download, public share |
| External service invocation | **Settings > Agent API** | Fixed-identity agent, deployment, API key | `/v1` runtime calls |

## Create a document analysis agent

Use this workflow when an agent needs to summarize long documents, compare tables, turn meeting notes into tasks, or review policy drafts.

1. Confirm the model and user credential in **Models** and **Credentials**.
2. Create an agent from **New Agent** using the builder, a template, or manual setup.
3. Write instructions for scope, output format, unsupported assumptions, and evidence handling.
4. Attach document reading, search, or file generation tools/skills when available.
5. Test with a short sample before using a large document.
6. If the run creates files, inspect the artifact card and **Settings > Files**.
7. Before sharing, review messages, tool results, and generated files for sensitive content.

| Check | Good state |
| --- | --- |
| Model | Supports the context size and tool calling needed for the task |
| Instructions | Clearly define summary, omissions, next actions, and evidence style |
| Files | Preview opens and version/update time are visible |
| Sharing | Messages, tool results, and files are reviewed before public access |

## Automate a recurring report

Recurring reports combine an agent, a schedule, generated artifacts, and usage monitoring.

1. Create an agent for the report and write repeatable instructions.
2. Attach any data access tools, MCP tools, or skills.
3. Run the agent manually once and confirm the result format.
4. Create a one-time, interval, or cron schedule from agent settings or **Schedules**.
5. Check timezone, `next_run_at`, `max_runs`, and end conditions.
6. After a run, inspect the conversation, run history, output preview, and trace ID.
7. Use **Usage** to monitor requests, tokens, and estimated cost.

<Warning>
Schedules can run when nobody is watching. If an agent calls external APIs or creates public-facing files, review run history and generated artifacts closely during the first few days.
</Warning>

## Extend an agent with MCP tools

MCP servers connect external tool collections to Moldy agents.

1. Open **Capabilities > MCP Servers**.
2. Choose `streamable_http`, `sse`, or `stdio`.
3. For URL transports, enter the URL and header credential. For `stdio`, enter command, args, and env vars.
4. Run **Probe** or **Test**.
5. Run **Discover** to import tool descriptors.
6. Select only the needed MCP tools in agent settings.
7. Test a small chat request and inspect tool cards, approvals, and failures.

MCP failures usually come from URL/command, credential templates, user credential ownership, network access, or model tool-calling support. See the [error reference](/hancom/moldy/en/error-reference) for the decision table.

## Review and share generated files

When a document skill or file tool creates DOCX, XLSX, PPTX, PDF, Markdown, HTML, or other outputs, users can open the file in chat and later find it in **Settings > Files**.

1. Open the artifact card in chat.
2. Confirm whether the format supports preview.
3. If the file changed multiple times, check version and update time.
4. Mark important files as favorites.
5. Download the current version when needed.
6. Before creating a share link, review messages, tool results, and file previews.

| Purpose | Best screen |
| --- | --- |
| Review a just-created file | Chat artifact card and right artifact panel |
| Compare files across conversations | **Settings > Files** |
| Review before public sharing | Share preview, artifact preview, public share artifact endpoint |

## Call an agent from an external service

For external calls, keep browser session APIs and Agent API runtime calls separate.

1. Confirm that the agent is a deployment candidate in **Settings > Agent API**.
2. Create a deployment and copy the public ID.
3. Create a server-side API key with scopes such as `invoke`, `stream`, or `read`.
4. Store the key in server-side secret storage. The cleartext key is shown only once.
5. Send a small `/v1/runs/wait` or `/v1/runs/stream` request.
6. During operations, keep key name, key prefix/last four, deployment public ID, run ID, and trace together.

See the [API Cookbook](/hancom/moldy/en/api-cookbook) for concrete examples.
