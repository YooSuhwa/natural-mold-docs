Moldy chat renders more than plain text. Markdown, code, diagrams, math, images, search results, tool runs, approvals, user input, memory proposals, sub-agents, Builder steps, and generated files can each appear as dedicated components.

For the full conversation workflow, see [Chat with an agent](/hancom/moldy/en/chat-with-agent). For streaming event structure, see [Chat streaming API](/hancom/moldy/en/api-chat-streaming). This page is a **reference for identifying UI that appears inside chat**.

## Component Matrix

| Component | When it appears | User action | Caution |
| --- | --- | --- | --- |
| Markdown body | General answers, explanations, summaries | Read headings, tables, and links | Tables can look incomplete while streaming |
| Code block | Code, logs, configuration examples | Check language label and content | Verify runnable code separately |
| Mermaid | `mermaid` code blocks | Inspect diagram structure | Syntax errors may stay as code |
| KaTeX math | Formula or calculation explanation | Check result and unit | Recalculate business-critical math |
| Inline image | Generated image, attachment, tool result | Expand or download | Review visual sensitive data before sharing |
| Plan card | Todo or staged work | Watch active and completed items | If stuck, inspect trace and tool state |
| Search result card | Search tool call | Check title, URL, and snippet | Judge external source trust separately |
| File tool card | File read/write/edit | Check target file and result | Confirm path, permission, overwrite behavior |
| Generic tool card | Tool without dedicated UI | Inspect input, status, result, error | Card errors can be more precise than final text |
| Approval card | Tool, prompt, or recommendation approval | Approve, reject, or edit JSON | Expiry can prevent resume |
| User input card | Clarifying question or HITL step | Type or choose an answer | Wrong input changes the remaining run |
| Memory card | Memory proposal or save event | Review before saving | Do not approve sensitive content |
| Sub-agent card | Delegated task | Check delegate name and result | Inspect the parent trace when it fails |
| Deep research summary | Research-style task result | Check sources and summary structure | Review source trust and recency |
| Builder card | Agent creation builder | Approve recommendations, image, draft config | Review settings before final confirmation |
| Artifact card | Generated file record | Preview, favorite, download | Check that status is `ready` |

## Builder Step Cards

| Card | Purpose | Check |
| --- | --- | --- |
| Phase timeline | Shows the current conversational creation phase | Completed and active phases |
| Recommendation review | Suggests tools to attach to the agent | Unneeded tools and permission scope |
| System prompt approval | Reviews the generated system prompt | Role, constraints, output format |
| Image choice | Chooses from candidate images | Brand fit and safety |
| Image approval | Confirms the selected image | Whether it can be shared publicly |
| Draft configuration preview | Reviews model, tools, skills, schedules, middleware | Missing settings and excessive permissions |
| Final configuration approval | Last checkpoint before creation | Name, description, tools, prompt |

## Read Component State

| State | Meaning | Next action |
| --- | --- | --- |
| Streaming | Message and cards are still forming | Wait for completion |
| Waiting | Approval, user input, or tool response is required | Act inside the card |
| Complete | The result was incorporated into the response | Review summary, files, trace, usage |
| Failed | Tool, credential, model, MCP, or storage failed | Inspect error card and trace first |
| Expired | Approval or input window passed | Retry with a new message or check resume |

## Approval and Memory Review

| Card | Check |
| --- | --- |
| Approval card | Target tool, input JSON, external calls, file changes, sharing, credential use |
| User input card | Question intent, choice meaning, effect on later execution |
| Memory card | Accuracy, sensitivity, storage scope, short editable phrasing |

## Files and Artifact Cards

Artifact cards are the closest entry point for files generated during a conversation. File-type preview providers and fallbacks are covered in the [Artifact viewer matrix](/hancom/moldy/en/artifact-viewers), while file-library workflows are covered in [Files and artifacts](/hancom/moldy/en/files-artifacts).

| Check in the card | Why |
| --- | --- |
| File name and extension | Explains which preview provider is selected |
| Status | Preview/download may fail unless status is `ready` |
| Source tool call | Shows which tool created the file |
| Version | Shows whether the file was regenerated |
| Share state | Shows whether it can appear in a public link |
