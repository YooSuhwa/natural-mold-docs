Moldy selects an artifact preview provider from the file `kind`, extension, MIME type, and status. Use this page as the reference for **which file opens in which viewer**.

For file library usage, favorites, downloads, and sharing workflows, see [Files and artifacts](/hancom/moldy/en/files-artifacts).

## Full Matrix

| File/content | Selection signal | Viewer | Check | Fallback/caution |
| --- | --- | --- | --- | --- |
| Images | `image/*`, `png`, `jpg`, `jpeg`, `webp` | Image preview | Expansion, visual content, download | Review visual PII before sharing |
| Audio | `audio/*`, `mp3`, `wav` | Browser audio player | Playback and duration | Browser codec support matters |
| Video | `video/*`, `mp4`, `webm` | Browser video player | Playback, poster, size | Large files may be better downloaded |
| PDF | `application/pdf`, `pdf` | PDF embed | Pages and document body | Browser PDF policies can affect preview |
| HTML | `text/html`, `html`, `htm` | Sandboxed iframe | Rendered result and links | Review external links and embedded media |
| Markdown | `text/markdown`, `md`, `markdown` | Markdown renderer | Headings, tables, links | Raw text fallback |
| Mermaid | `mmd`, `mermaid` | Mermaid renderer | Flowchart and sequence output | Syntax errors may remain as code |
| CSV/TSV | `csv`, `tsv`, text table | Table preview | Delimiter, header, row count | Large files may show a partial view |
| JSON | `application/json`, `json` | Structured JSON preview | Keys and array/object depth | Review sensitive fields |
| YAML/TOML | `yaml`, `yml`, `toml` | Key/value preview | Config values and placeholders | Review secret-like values |
| Code | artifact kind `code` | Code preview | Language, path, overwrite risk | Verify before running |
| TXT/LOG | `text/*`, `txt`, `log` | Text preview | Error lines and stack traces | Review internal paths and tokens |
| DOCX | `docx` | Document preview | Body, tables, headings | Download fallback |
| XLS/XLSX | `xls`, `xlsx` | Sheet/table preview | Sheets, rows, columns | Check formulas and hidden sheets |
| PPTX | `pptx` | Slide preview | Slide order and body | Download fallback |
| HWP/HWPX | `hwp`, `hwpx` | Korean document preview | Paragraphs, tables, body structure | Download fallback |
| Other/CAD | `cad`, `dwg`, unknown MIME | Fallback card | File name, size, download | Consider adding a preview provider |

## Selection Inputs

| Input | How Moldy uses it |
| --- | --- |
| `kind` | Broad category such as `image`, `video`, `audio`, `pdf`, `markdown`, `html`, `code`, `document`, `data`, `cad`, `other` |
| Extension | File-name signal. Important when MIME type is incomplete |
| MIME type | Content type such as `image/*`, `text/html`, `application/pdf`, `application/json` |
| Status | Preview/download should be available when status is `ready`; `writing`, `failed`, and `deleted` are handled as separate states |
| Size limits | Text preview byte limit, maximum artifact bytes, and maximum files per run can affect display |

## Location Differences

| Location | Best for |
| --- | --- |
| Chat artifact card | Recently generated file and run context |
| Right artifact panel | Files in the current conversation |
| **Settings > Files** | Search, filters, favorites, stats across conversations |
| Public share screen | Read-only snapshot and shared files by share token |

## Failure Checks

| Symptom | Check first |
| --- | --- |
| Empty preview | Artifact status, MIME type, extension, size limit |
| Download fails | Storage object, current version, owner permission |
| Public share fails | Active share token and artifact-conversation relation |
| Only fallback appears | Extension/MIME provider support |
| Old file appears | Version number and update time |

<Warning>
HTML previews open in a sandboxed iframe. Still review external links, embedded media, and sensitive body content before sharing.
</Warning>

## Maintenance Checklist

When viewer behavior changes, update:

1. Extension/MIME rows in this matrix.
2. The summary table in [Files and artifacts](/hancom/moldy/en/files-artifacts).
3. Artifact card notes in [Chat component gallery](/hancom/moldy/en/chat-components).
4. Public sharing and security notes.
5. Files/artifacts entries in `src/llms.txt`.
